# ADR-001: Стратегия запуска WF-2

**Дата:** 11.06.2026  
**Статус:** Принято  
**Автор:** Veniamin Savchuk  

## Контекст

WF-2 (Балансировка покрытия) должен запускаться после полного завершения WF-1,
поскольку ему нужны актуальные данные FTE, пересчитанные после загрузки пакета WF-1.

## Рассматривались варианты

1. **Отдельный Cron с запасом** — WF-2 стартует по таймеру через X минут после WF-1.
   - Риск: длительность WF-1 зависит от числа агентов (~2000), запуск «с запасом» ненадёжен.

2. **Execute Workflow / внутренний webhook** — WF-1 программно запускает WF-2 по завершении.
   - Гарантирует порядок выполнения независимо от времени работы WF-1.

## Решение

Использовать **n8n Execute Workflow** (sub-workflow): WF-1 в последнем шаге вызывает WF-2,
передавая контекст (дата смены, run_id). WF-2 не имеет отдельного Cron-триггера.

## Последствия

- WF-2 никогда не запустится без WF-1 в тот же день (это ожидаемое поведение).
- При сбое WF-1 на этапе загрузки (BE4) WF-2 не запускается — просадки дня остаются.
- В n8n воркфлоу WF-2 должен иметь триггер типа `Execute Workflow Trigger`.
