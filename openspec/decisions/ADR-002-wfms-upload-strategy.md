# ADR-002: Стратегия загрузки изменений в WFMS

**Дата:** 10.06.2026 (уточнено 11.06.2026)  
**Статус:** Принято  
**Автор:** Veniamin Savchuk  

## Контекст

Изменения расписаний нужно загружать в WFMS (`wfms.avito.ru`) через web UI.
Объём WF-1: ~2000 агентов в день. Исходный вариант — загрузка по 1 агенту с
пересчётом FTE после каждого — занял бы ~16–17 часов (неприемлемо).

## Рассматривались варианты

**WF-1:**
1. Загрузка по 1 агенту с пересчётом FTE после каждого → ~16–17 ч. **Отклонено.**
2. Загрузка одним файлом-пакетом (все агенты) + однократная публикация и пересчёт FTE. **Принято.**

**WF-2:**
1. Загрузка одним пакетом всех групп без пересчёта FTE между группами → группы независимы, пересчёт в конце. **Принято** (реш. 11.06).

## Решение

- **WF-1:** Накапливает строки всех агентов в пакет, загружает одним файлом в оргточке «Поддержка», затем однократно «Опубликовать изменения» → «Пересчитать FTE». В пакет попадают только агенты с фактическими изменениями.
- **WF-2:** Загружает перемещения пакетом в конце, внутри пакета — по 1 сотруднику с «Опубликовать» + «Пересчитать FTE» после каждого.

## Последствия

- Необходимо уточнить у WFM лимиты batch-загрузки (размер файла / число строк) — **открытый вопрос Q-02**.
- Checkpoint (запись в БД) обязателен: при сбое на загрузке — не пересчитывать уже загруженных агентов.
- WF-1 не контролирует покрытие FTE во время расчёта — это задача WF-2 после пересчёта.
